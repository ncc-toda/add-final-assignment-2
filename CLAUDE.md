# CLAUDE.md

天気予報TUIツール「weather」(Rust製)の開発リポジトリ。

## ドキュメント

- 要件定義: `docs/requirements.md`(技術スタック・機能要件・非機能要件の正)
- 実装計画: `docs/plans/mvp_plans.md`(タスク一覧・進捗・依存関係)

作業前に関連タスクの計画を必ず確認すること。技術スタック等の詳細はここには再掲しない。

## 開発フロー

- 各タスク着手時は `grilling` skillで詳細設計(データ構造・関数分割・テストケース等)を詰めてから実装する
- ロジック実装はt_wada式TDD(Red→Green→Refactor)を厳守する。TUI描画部分は手動確認とし自動テスト対象外
- 仕様・技術選定に変更が生じた場合は `docs/requirements.md` も合わせて更新する

## Planファイル運用

- 実装計画ファイルは `docs/plans/` に格納する
- 全タスクが完了したplanファイルは `docs/plans/done/` に移動する

## Git運用

- mainブランチに直接commitする(ブランチ運用・PRレビューは行わない)
- コミットメッセージは日本語で記述する

## 完了報告

作業完了時は以下を簡潔に報告する:

1. 対応内容(2, 3行程度)
2. 主な変更ファイル
3. 使用したツール(skills, mcp)
4. 次やるべきこと
